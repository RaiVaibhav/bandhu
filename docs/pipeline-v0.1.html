Bandhu — Engineering Draft

Check-in pipeline, v0.1

One user check-in, end to end. Hybrid architecture: safety, tagging, retrieval, and guardrails are fixed and deterministic; a single orchestrator call holds the only real judgment calls — the three things the product spec treats as optional and contextual.

Deterministic step — always runs, no model discretion
LLM judgment — orchestrator tool-call decision
Async / sampled — doesn't block the response
Crisis branch
Special-case redirect

Flow diagram

10 stages · 3 branches
1
Deterministic

Ingest & normalize

Text, voice-to-text, image caption, or a mood-tap value. Detects language (incl. code-mixed). Image inputs get a document-type check before anything else touches them.

2
Deterministic

Safety gate

Rules-based crisis-language scan. Reads current message + recent conversation context, not the isolated turn — so a hedge after a concerning phrase can't suppress it.

3
Deterministic

Classify

Two parallel checks: special-case flag (medical / legal / disorder / dependency / medication / minor-mode), and general emotion·category·intensity tag for retrieval.

4
Deterministic

Memory read

Light structured summary only — recurring themes, last suggestion offered, whether it helped. Never raw transcript.

5
Deterministic

Retrieval

Metadata-filtered pgvector query on the emotion tag. Top 2-3 chunks. Runs whenever there's a tag — doesn't guarantee the result gets used.

6
LLM judgment

Orchestrator

The only discretionary step. Picks at most one of three tools per turn, to protect the word budget and the "quiet, easy to ignore" tone.

offer_suggestion? name_thinking_trap? close_the_loop?
7
Deterministic

Generate

Fast/cheap model phrases: acknowledgment + whichever single optional element the orchestrator chose. Hard-capped ~60 words. Content only from retrieved chunks — never freely generated advice.

8
Deterministic

Guardrail check

Length, banned diagnostic/medical/legal phrasing, no-advice-giving. Rules-based, not the generation model grading itself.

9
Deterministic

Memory write → response

Update the structured summary (mood, theme, suggestion offered), then return the response to the user.

Crisis response

Fixed template — real numbers, Call Now / WhatsApp. Bypasses every remaining stage. Shown once per conversation; a repeat true-positive doesn't re-trigger the card, but a hedge never suppresses the first one.

Special-case redirect

Fixed, vetted template per category — rejoins the spine at Guardrail check, skips the orchestrator's discretion entirely.

  • Medical doubt → doctor's-notes export
  • Life decision → reflect, don't recommend
  • "Do I have X" → what an evaluation involves
  • Dependency signal → warm, plain redirect to people
  • Medication question → hard redirect, no exceptions
  • Medical document image → declined, redirected

Fallback safe response

Guardrail fail → generic acknowledgment only, no retrieved content, no optional element. Logged for review.

Sampled evaluator — async, 5–10%

MITI-based LLM judge scores tone on a sample of responses, after the user already has theirs. Feeds Phoenix for quality monitoring, not gating.

What the stress test surfaced

6 findings
Blocks build

Safety gate needs conversation memory

Scoring the latest message alone misses the "I'm fine, just thinking about it" hedge case the spec calls out. Needs a small rolling window of recent turns, plus a per-conversation "already shown" flag so a confirmed true positive doesn't re-trigger the crisis card on every later message.

Blocks build

Six special-case redirect templates, not yet written

Medical doubt, life decisions, disorder questions, dependency signals, medication questions, and medical-document images each need a vetted, fixed template — these bypass the orchestrator entirely, so they can't be freely generated. This is a content-library gap, feeding straight into the resources doc.

Blocks build

Image ingest must reject medical documents before captioning

The spec's "no document-upload pathway feeds the AI" needs to be a check that runs at ingest, on every image, before anything downstream sees it — not just an instruction to the model.

Decision needed

Minor / age-unknown is a session flag, not a message tag

It has to gate the safety gate threshold, the guardrail strictness, and which crisis resource list gets shown (e.g. a minor-appropriate helpline) for the whole session — not be re-derived per message. README already flags the underlying policy as unresolved; the flow just needs the plumbing ready for whichever way that lands.

Decision needed

Orchestrator needs a one-tool-per-turn rule

The exam/breathing test case wants both close_the_loop and offer_suggestion at once. Stacking both risks blowing the ~60-word cap and turning "one small offer" into a list. Needs an explicit priority order (loop-closing likely outranks a fresh suggestion) rather than leaving it to the model to decide how much is too much.

Decision needed

Low-confidence tags need an honest fallback

"idk" or a bare emoji shouldn't get force-fit into the nearest emotion category just so retrieval has something to return. Classify should be able to say "not enough signal" and have the orchestrator default to a plain open acknowledgment.

v0.1 — draft for discussion, not implementation. Next: fold the six special-case redirect templates and the safety-gate context window into bandhu-research-resources.md, then define the tone rubric each stage is actually graded against.
